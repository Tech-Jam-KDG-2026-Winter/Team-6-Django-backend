Background & Issues

Why
We Built It

GUIの欠如

手元のサンプル音源を即座に加工・確認できる使いやすいGUIが存在せず、コマンドライン操作等の手間が発生していた。

複数ツールの分断

音声編集、フォーマット変換、エフェクト適用で複数のツールを行き来する必要があり、作業効率と生産性が著しく低かった。

Concept & Approach

How
We Solved It

Djangoによるワンストップ化

WebフレームワークDjangoでシステムを集約。REST APIとプリセット機能の実装により、UIから即座にエフェクトを適用可能に。

FFmpeg + Numpyの自動化

FFmpegでフォーマット差異を吸収し、Numpyによる計算処理で正規化（ノーマライズ）を自動化。複雑なエンジニアリングを裏側で処理。

Workflow & Features

サイトの使い方

User Flow
01

Upload / Input

マイクからの直接入力を受け付けます。即座に準備中の表示が出てきます。

02

Select Preset

「Clean」「Radio」「Robot」などのPedalboardプリセットを選択。複雑なエフェクトチェーンを一括適用します。

処理後の音声を自動で正規化（ノーマライズ）。最適な音量レベルに調整して出力します。

Key Features

Chunk Processing

長尺音源もチャンク分割処理により、メモリ負荷を抑えたストリーミング対応を実現。

FFmpeg Conversion

入出力確認とバックエンドでのFFmpeg連携により、多様なフォーマット変換を自動化。

Admin Access

スタッフ権限を持つユーザーは、全ユーザーの処理履歴とファイルを一元管理可能。

Implementation Details

特に力を入れたポイント

音声処理のリアルタイム性と品質を両立させるため、バックエンドの設計にこだわりました。特に低遅延化と安定性を確保するための技術選定と実装がプロジェクトの核となっています。

6
Key Technical Challenges

User friendly UI

UIの簡素化を図り、ユーザーが操作しやすいようにしました。

Effect applied audio output

どのファイルを選択しているかわかりづらくなってしまっていたので、シンプルな音声がアウトプットされるようにしました。

Audio Normalization

録音環境などから、音量がばらついてしまっていたので、音量を一定に保つようにしました。

Pedalboard Chain

Gain, Chorus, Reverb, Compressor等を組み合わせ、用途に応じた最適なエフェクトチェーンを設計しました。

REST Endpoint

UIからの操作を待たせないよう、非同期で処理する仕組みにしました。

Storage Management

DjangoのFileFieldとSampleモデルを活用し、処理前後の音声を、無駄なく整理して保存できるようにしました。

Retrospective

制作振り返り

ツール集約の達成

複数のツールを行き来していた作業フローをDjangoアプリに集約。アップロードから加工までシームレスな体験を実現しました。

処理の抽象化

音声I/Oとエフェクトチェーンをモジュール化・抽象化することで、将来的な機能拡張やプリセット追加が容易な設計となりました。
